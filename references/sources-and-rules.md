## Recurring Partnership Events

Monitor these annual or recurring events even when the current edition is not announced:

- 臺灣科學節：通常在 11 月；適合 STEAM、科學、創客與體驗工作坊，通常以合作或策展形式參與。

- 台灣設計展：通常在秋季並輪流於不同縣市舉辦；適合設計教育、兒童體驗與跨域內容合作。
- 台北國際旅展：通常在 10–11 月；僅在產品涉及戶外學習、營隊、遊學或教育旅行時納入。

Clearly label these as `合作型參與` unless a formal exhibitor application is verified.

Also check official education bureaus, cultural affairs bureaus, libraries, exhibition venues, organizer associations, and government press releases.

## Age and Learning Relevance

The primary age scope is 3–12:

- 3–6: preschool, early literacy, play-based learning, sensory development, parent-child learning
- 6–9: lower elementary, literacy, numeracy, science exploration, arts, social-emotional learning
- 9–12: upper elementary, STEAM, coding, inquiry, project learning, reading, creativity

Record the applicable age band and learning domains. Exclude infant-only, teen-only, university-only, and adult-only events unless a substantial 3–12 section exists.

## Source Ranking

1. Organizer's official event page or exhibitor prospectus
2. Government, venue, or industry association announcement
3. Organizer's official social account
4. Ticketing or event-listing platform
5. News report

Use lower-ranked sources only for discovery or when clearly labelled as unconfirmed.

## Change Detection

Compare the newly verified record with the existing tracker. Flag changes to:

- Event dates
- Exhibitor registration opening or deadline
- Venue
- Cancellation or postponement
- Official website
- Attendance figure

Update `最後更新日` only when the row is checked. Describe meaningful changes in `備註`.

## Previous-Edition Planning

For every tracked event, search the organizer's archive, previous news releases, venue calendar, and event posters for the immediately preceding edition.

- Record verified previous-edition start and end dates in dedicated columns.
- Record the source URL or `待查`.
- Never copy previous-edition dates into current-edition date fields.
- Use approximately 180 days before the previous start date as the default planning-start reference.
- For first-time events or events without a verifiable previous edition, leave dates blank.
